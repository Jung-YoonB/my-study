#### 캡슐화

- 정보 은닉 특성
    - 데이터가 함부로 변경되지 않도록 접근 제한자를 private로 숨긴다.
    - 단, 데이터를 활용한 기능은 외부에서 사용이 가능하도록 public으로 열어둔다.
        - getter / setter

---

#### 상속

- 다른 클래스가 가지고 있는 멤버들을 새로운 클래스가 물려받을 수 있는 특성
    - 멤버?
        - 데이터(변수/필드), 메소드
    - 직접 정의 하지 않고 사용 가능
    - 모든 클래스는 object 클래스의 자식
    - 부모 클래스의 생성자, private 멤버는 상속 X
    - 자바에서는 단일 상속만 지원 가능
    - extends 키워드를 사용해 상속 가능
        
        ```java
        접근제한자 class 클래스명 extends 부모클래스명 {}
        ```
        
    
- 오버 라이딩을 통해 부모 클래스의 메소드를 재정의 할 수 있다.
    - 단, 메소드명, 매개변수 정보, 반환형이 모두 같아야 함.

---

#### 다형성

- 하나의 참조 변수가 여러 형태를 가질 수 있는 성질
    - 정적 바인딩
        - 참조 타입과 인스턴스(객체) 타입이 일치할 때 컴파일 단계에서 접근 가능
        
    - 동적 바인딩
        - 컴파일 단계
            - 참조 타입 메소드와 연결(정적 바인딩)
        - 실행 단계
            - 재정의 된 메소드(오버 라이딩) 호출
    
    - 업 캐스팅
        - 부모 클래스 타입의 변수로 자식 클래스 객체를 참조 가능
            - 자식 타입 → 부모 타입 클래스로 자동 형 변환
    
    - 다운 캐스팅
        - 부모 타입으로 참조한 자식 클래스를 다시 자식 클래스 타입으로 강제 형 변환 하는 것
        - 단, instanceof 연산자를 통해 타입 확인 후 형 변환 해주어야 함.
            
            ```java
            참조변수 instanceoh 확인할타입(클래스명)
            
            * boolean 타입으로 결과 반환
            
            if (참조변수 instanceoh 확인할타입(클래스명)) {
            	 ((확인한타입_클래스명)참조변수).getter();
            }
            
            조건문 체크로 위와 같이 사용 가능
            ```
