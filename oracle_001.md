### 관계형 데이터베이스(RDB)

- 데이터를 표 형태로 저장하고, 관계를 통해 데이터의 연결성과 정합성을 유지하는 데이터베이스
- 구성 요소
    - 테이블
        - 데이터를 저장하는 기본 단위
    - 행 (튜플)
        - 실제 데이터 1건
    - 열 (컬럼/도메인)
        - 데이터의 속성 (항목)
    - 기본키 (Primary Key)
        - 각 행을 식별하는 값
    - 외래키 (Foreign Key)
        - 다른 테이블의 기본키를 참조하는 값

---

### NoSQL

- 비관계형 데이터베이스
    - 고정 된 테이블 구조 X
    - 대용량 데이터 처리, 수평 확장
    - 빠르게 변화하는 데이터 구조에 적합
    - 속도 / 유연성 / 확장성 최우선 고려
- 대표 유형
    - 문서형
        - JSON/BSON : 블로그 포스트, 상품 정보
    - 키-값형
        - Key - Value : 세션 관리, 캐시
    - 열 (Column) 지향
        - 열 단위로 저장/조회 : 로그 수집, 시계열 데이터
    - 그래프형
        - 노드-간선 구조로 관계 표현 : SNS 친구 관계, 추천 시스템
