**클래스 기초 개념**

- 클래스란?
    - 추상화를 통해 객체의 특성을 정의해 놓은 설계도
    - 추상화 : 프로그램에 필요한 정보 공통점을 추출하고, 불필요한 특성을 제거하는 마인드맵 과정

- 객체(인스턴스)
    - 클래스(설계도)를 바탕으로 heap 메모리에 할당된 결과물

- 클래스의 구성
    - 필드
        - 객체를 구성하는 변수 선언
    - 생성자
        - new 연산자를 사용, 객체를 생성할 때 호출하기 위한 특별한 메소드
        - 아무 생성자를 생성하지 않을 경우, 기본 생성자가 생성
    - 메소드
        - 함수 : 클래스 내에서 특정 기능을 묶어 놓은 코드블럭
        - 표현법
            [접근제한자] [예약어] 반환형 메소드명([매개 변수]) {
            		// 실행 할 내용
            }
            * [대괄호 안은 생략 가능]
                   

- 접근 제한자
    - public
    - protected
    - default ⇒ 생략 가능
    - private

- 반환형 ⇒ 생략 불가
    - 반환값 X : void
    - 반환값 O : 자료형 으로 반환형 작성

- 매개 변수
    - 전달 받을 값 X : 생략 가능
    - 전달 받을 값 O : (자료형 변수명) 으로 매개 변수 작성

- 메소드 오버로딩
    - 하나의 클래스 안에서 동일한 이름의 메소드를 여러개 만들 수 있음.
    - 단, 아래 세가지의 매개변수 정보 중 하나는 반드시 달라야함
        - 매개 변수의 개수
            - (int a)
            - (int a, int b)
        - 매개 변수의 타입
            - (int a)
            - (string a)
        - 매개 변수의 개수와 타입이 2개 이상일 경우 순서
            - (int a, String b)
            - (String a, int b)
